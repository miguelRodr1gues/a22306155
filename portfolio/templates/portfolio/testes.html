Testes da Aplicação

Introdução aos Tipos de Teste

No desenvolvimento de software, os testes são essenciais para garantir a qualidade e o funcionamento correto da aplicação. Existem vários tipos de testes, entre os quais destacam-se:

  • Testes unitários: verificam o comportamento correto de unidades isoladas de código, como funções ou métodos.
  • Testes de integração: validam a interação entre diferentes componentes ou módulos da aplicação.
  • Testes funcionais ou de sistema: verificam o funcionamento da aplicação como um todo, simulando o uso real.
  • Testes end-to-end: testam toda a aplicação do início ao fim, incluindo interface, lógica e base de dados.

Testes de Integração Desenvolvidos

Foram criados testes que verificam o funcionamento integrado de modelos, URLs e views, assegurando que as principais rotas e páginas da aplicação funcionam corretamente e exibem os dados esperados.

Os testes confirmaram que:

  • As URLs estão bem configuradas e direcionam para as views certas;
  • As views retornam o código HTTP 200 com os templates corretos;
  • Os conteúdos importantes são exibidos nas páginas.

Exemplo de Teste de Integração

import pytest
from django.urls import reverse
from django.test import Client

@pytest.mark.django_db
def test_artigo_detalhe_view():
    client = Client()
    response = client.get(reverse('artigo_detalhe', args=[5]))
    assert response.status_code == 200
    content = response.content.decode()
    assert "NEGAO" in content
    assert "Bernardo Jerónimo" in content
    assert "artigos/artigo_detalhe.html" in [t.name for t in response.templates]

Cobertura dos Testes

Para medir a cobertura dos testes, utilizei o pacote coverage seguindo estes passos:

  1. Instalar o pacote com pip install coverage;
  2. Executar os testes com cobertura usando coverage run manage.py test;
  3. Visualizar o relatório no terminal com coverage report;
  4. Opcionalmente, gerar relatório visual em HTML com coverage html e abrir o ficheiro htmlcov/index.html no navegador.

A cobertura obtida foi satisfatória, garantindo que as partes principais da aplicação estão testadas e a funcionar conforme esperado.
